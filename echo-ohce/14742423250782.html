move cursor to previous marked location

Every edited location is an automatically marked location

Command Description
ma set mark a at current cursor location
'a - quotemark jump to line of mark a (first non-blank character in line)
`a - backtick jump to position (line and column) of mark a
'' - 2 quotemark jump to the head of the line of previous location
`` -2 backtick jump to the previous character location
'. - quotemark + dot jump to the head of the line of previous edited location
`. - backtick + dot jump to the previous edited character location
:marks list all current marks (including marks of other files)
:jumps list all current jumplist (including jumps of other files)
:help mark-motions help for mark related stuff
:help jump-motions help for jump related stuff

Jump to previous jumplist locations

This is different than move, because:
1. It is not set by markers, It is automatically saved inside the jumplist
2. Not all moving actions count as jump. The detailed jump action list is described in :help jump:

A "jump" is one of the following commands: "'", "`", "G", "/", "?", "n",
"N", "%", "(", ")", "[[", "]]", "{", "}", ":s", ":tag", "L", "M", "H" and
the commands that start editing a new file

Jump can jump multiple counts
It uses:
Ctrl + o to jump backward in the jumplist
5 + Ctrl + o to jump backward five times in the jumplist
Tab to jump forward in the jumplist
3 + Tab to jump forward three times in the jumplist.

Recording repeatable actions

The most useful usecase is .
Vim automatically records the last action (change made), use
. can repeat that action. Reference

More complicated actions, can be recorded using q + [a-z] the letter [a-z] kind of give the recorded actions a name. Press q again in the normal mode, stops the recording. @ + [a-z] repeats the actions recorded, and @@ repeats the latest recorded actions.

Visual and Visual Block

Visual is just v, which is useful for selecting and copying.

Visual Block is by Ctrl + v, which is useful for bulk edit. It kind of copy the same actions to all the lines selected by the Visual Block. You have to press Esc then see the actions to be copied and applied to all the lines. Best usecase is adding comments to a block of code

Visual Block best uses with:
i or I to insert stuff on all lines at the same vertical position
c or C to change or add stuff on all lines at the same vertical position

Editing multiple files

  1. :e <filename> opens another file in buffer
  2. :e . is the same as ls, but showing it as a file
  3. :ls list all the buffers.
  4. :bd delete current buffer, i.e. close it
  5. :b <bufferid> jump to another buffer
  6. :bn jump to next buffer
  7. :bp jump to previous buffer
