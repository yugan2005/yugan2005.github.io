SSH connections

2017/7/3 posted in  tips

Set the ssh from MacBook to linux desktop

  • seas MP must be build on Linux box, so we have to able to ssh to desktop
  • Enable desktop ssh Server Reference:
    • start sshd from command line: service sshd start
    • start daemon automatically: chkconfig sshd on

  • Set the public and private key pairs

    • scp the ygan_at_linkedin.com_ssh_key.pub(public) from MacBook to Linux box and save it @ ~/.ssh/authorized_keys, change mode to 600 reference

    • The following are for avoiding enter passphrase every time

      • Set the following lines in .bash_profile on both the MacBook and the linux box reference
      if [ -x /usr/bin/keychain ] ; then
  MYNAME=`/usr/bin/whoami`
  if [ -f ~/.ssh/${MYNAME}_at_linkedin.com_ssh_key ] ; then
        /usr/bin/keychain ~/.ssh/${MYNAME}_at_linkedin.com_ssh_key
              . ~/.keychain/`hostname`-sh
  fi
fi
      • Every session (Maybe every reboot?) add the identity to the ssh-agent:
      # starting the ssh-agent
eval $(ssh-agent) 

# add the private key to ssh-agent. Will be asked for passphrase once for all.
ssh-add ~/.ssh/ygan_at_linkedin.com_ssh_key

The final solution for vnc (remote desktop)

  • Mac use system default app screen sharing
  • Linux (redhat 6) installed:sudo yum install tigervnc-server
  • (maybe optional) Disabled all firewall on Linux sudo service iptables stop
  • I setup a separate user #2. Every time after system reboot need run this on Linux: vncserver :2
  • The connection on Mac is ygan-ld1.linkedin.biz:5902
